🧬 Detail Teknis Implementasi

Saya mendesain sistem dynamic routing untuk meng-handle optimasi landing page lokal di 100+ kota Indonesia. Fokus utama ada pada dua template endpoint yang didefinisikan secara programatis menggunakan constraint regular expression di Laravel:

// 1. Endpoint Harga Lapangan per Kota Route::get('/jual-karpet-{sport}-{kota}', [KarpetKotaController::class, 'showJual']) ->name('karpet.jual.kota') ->where('sport', 'badminton|pickleball|basket|minisoccer') ->where('kota', '[a-z\-]+'); // 2. Endpoint Halaman Jasa Pemasangan per Kota Route::get('/layanan/lapangan-{sport}-{slug}', [KotaLayananController::class, 'show']) ->name('kota') ->where('sport', 'badminton|padel|basket|minisoccer|pickleball');

Hasil: Mengurangi overhead pemeliharaan link. Cukup menggunakan 2 definisi route ini untuk secara otomatis memetakan, mem-build, dan me-render 400+ landing page dengan performa loading instan.

Laravel Routing Regex Constraints Clean URL Structure

Bagaimana menampilkan portofolio relevan di kota yang belum pernah ditangani? Saya merancang algoritma pencarian bertingkat (cascading fallback) 6 tahapan di level controller:

// 1. Cari kecocokan proyek di database untuk kota spesifik $proyek = Proyek::active()->bySport($sport)->where('slug', $slug)->first(); // 2. Jika tidak ada, deteksi wilayah (misal Jabodetabek cluster) if (!$proyek && $isJabodetabek) { $proyek = Proyek::active()->bySport($sport)->whereIn('kota', $jabodetabekList)->first(); } // 3. Fallback bertahap: Satu Provinsi -> Satu Pulau/Regional -> Fallback Headquarter

Dampak: Halaman landing page kota-kota baru tidak pernah kosong (thin content) atau menampilkan error 404. Google mengindeksnya dengan sempurna karena halaman menyajikan data proyek terdekat yang bernilai tinggi bagi user lokal.

Eloquent Logic Fallback Architecture Crawl Budget Optimization

Untuk memastikan visual landing page bervariasi mengikuti jenis cabang olahraga yang diklik, saya menerapkan dynamic theme mapping yang disimpan dalam file config terpisah (`themes.php`):

// themes.php snippet return [ 'badminton' => [ 'heading' => 'text-orange-600', 'badge_bg' => 'bg-orange-50', 'btn' => 'bg-orange-600 hover:bg-orange-700', ], 'minisoccer' => [ 'heading' => 'text-emerald-500', 'badge_bg' => 'bg-emerald-50', 'btn' => 'bg-emerald-600 hover:bg-emerald-700', ] ];

Dampak: 1 template Blade tunggal dapat me-render visual tema yang berbeda-beda tanpa duplikasi file CSS atau file view, meminimalkan memory footprint dan merapikan codebase.

Configuration Mapping Design Token Reuse Tailwind CSS Dynamic

Demi kecepatan load yang maksimal, basis data SQLite digunakan karena sifatnya yang single-file dan ringan. Untuk menghindari query join yang rumit dan berat, schema proyek memanfaatkan model SQLite dengan column JSON untuk data FAQ, list area, dan asset metadata:

protected $casts = [ 'active' => 'boolean', 'stats' => 'array', // JSON data: [{"value":"6","label":"Lapangan"}] 'faqs' => 'array', // JSON data: [{"q":"...","a":"..."}] 'images' => 'array', // JSON data: {"thumb1":"...", "thumb2":"..."} ];

Dampak: Data kompleks terenkapsulasi dalam satu row record proyek. Laravel secara otomatis mengkonversinya menjadi array manipulatif saat dipanggil di Blade template.

SQLite Database Eloquent Casts Data Deserialization

Tanpa server CI/CD yang mahal, saya merancang bash script otomatis untuk deploy granular ke Hostinger Shared Hosting via SSH/SCP. Script ini memastikan transfer file berjalan cepat dan aman, dilengkapi backup SQLite database otomatis sebelum file baru ditimpa.

Dampak: Proses pembaruan sistem dari lokal ke live server hanya memakan waktu < 30 detik tanpa down-time, menjaga website tetap aktif melayani pengguna.
